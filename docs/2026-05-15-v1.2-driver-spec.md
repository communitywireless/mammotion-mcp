## The bug

`mammotion_mcp/tools/mow.py`'s `mow_area` tool fires `lawn_mower.dock` immediately after observing `state=mowing` when `mow_duration_sec=None` and `return_to_dock=True` (the defaults). Empirical: at 09:25 HST today, the dock fired 6 seconds after the mower entered MODE_WORKING — HA's mammotion integration translates a dock-during-mowing into `pause_execute_task` + `return_to_dock`, which killed the mow before the mower could navigate to Area 1.

v1.1's verification phase doesn't fix this — it delays the dock fire by the verification window. If verification passes (blade engagement confirmed for ~3.6s), the next line of code fires dock and aborts the mow it just verified started.

## The fix

Change the dock semantic in `mow_area`:

| `mow_duration_sec` | `return_to_dock` | v1.1 behavior | v1.2 behavior |
|---|---|---|---|
| None | True (default) | Fire dock immediately after state=mowing (BUG) | Do NOT fire dock. Mower auto-completes the area + auto-docks itself. Tool returns after v1.1 verification passes (Phase 3 blade-engaged confirmed). |
| None | False | Return after start_mow (no dock) | Same. |
| N (set) | True | Sleep N seconds, then fire dock (explicit recall) | Same — explicit user-requested recall preserved. |
| N (set) | False | Sleep N seconds, then return without docking | Same. |

The asymmetric semantic: **`mow_duration_sec=None` means "let the mower mow the entire area to natural completion."** The mower's own firmware auto-docks when the area is finished. `mow_duration_sec=N` means "mow for at most N seconds, then I'm recalling you."

## Specific code change shape

Current v1.1 logic in `mow_area` (paraphrased):

```python
# After v1.1 verification phase passes:
if mow_duration_sec:
    await asyncio.sleep(mow_duration_sec)

if not return_to_dock:
    return {result: "mow_complete", ...}

# THIS IS THE BUG — fires unconditionally after v1.1 verify passes:
await ha_client.call_service("lawn_mower", "dock")
await _poll_charging(...)
await ha_client.call_service("mammotion", "cancel_job")
return {result: "mow_complete", ...}
```

v1.2 fix:

```python
# After v1.1 verification phase passes:
if mow_duration_sec is None:
    # User wants natural completion. Mower auto-docks on its own.
    # DO NOT fire dock. Return success — verification confirmed mowing started.
    return {
        result: "mow_complete_autonomous",
        verification: verify_result,
        mower_status: await ha_client.get_mower_status().to_dict(),
        protocol_version: 3,
        note: "Mower will continue mowing + auto-dock when area complete. "
              "Did not fire explicit dock — pass mow_duration_sec to recall.",
    }

# mow_duration_sec is set — user explicitly bounded the mow with a recall
await asyncio.sleep(mow_duration_sec)

if not return_to_dock:
    return {result: "mow_dispatched_with_duration_no_recall", ...}

await ha_client.call_service("lawn_mower", "dock")
await _poll_charging(...)
await ha_client.call_service("mammotion", "cancel_job")
return {result: "mow_complete", ...}
```
